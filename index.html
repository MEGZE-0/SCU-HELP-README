SCU-HELP Documentation

Introduction

Welcome to the documentation for SCU-HELP. This document provides comprehensive instructions on how to set up, run, and test the project locally using Postman. SCU-HELP is a web application designed for Students To Help theme manage their files and materials.

Prerequisites

Before proceeding, ensure you have the following prerequisites installed on your system:

  • Node.js (with npm) (recommended version is 21.x)
  • Git
  • Visual Studio Code (optional but recommended)
  • Postman

Installation

Clone the repository:

git clone <repository-url>

Open the project in Visual Studio Code (optional):

code <project-directory>

Navigate to the backend directory:

cd backend

Install dependencies:

npm install

Running the Project

To run the project in development mode, use the following command:

npm run dev

Alternatively, you can run it using:

node app.js

The project will start running on http://localhost:3000 by default.

Accessing the Website

Once the server is running, you can access the website by opening a web browser and navigating to:

http://localhost:3000

Testing with Postman

  1. Ensure the backend server is running.
  2. Open Postman.
  3. Import the provided Postman collection file (<project-name>.postman_collection.json).
  4. Once imported, you will find a collection named SCU-HELP with various routes for testing.
  5. Before testing, ensure to set up the necessary cookies for authenticated routes. You can do this by:
    • Open any browser and install a cookie editor extension.
    • Log in to the website and copy the session cookie.
    • In Postman, navigate to the Cookies section of the request and replace the existing cookie with the copied one.
  6. Test the routes as needed.

Routes

  • GET /get-username: Retrieve username.
  • GET /get-userdata: Retrieve user data.
  • POST /admin-register: Register admin.
  • GET /admin/users: Get list of admin users.
  • GET /admin/documents: Get admin documents.
  • DELETE /admin/document/:documentId: Delete admin document.
  • POST /forgot-password: Request password reset.
  • GET /reset-password: Reset password.
  • POST /reset-password: Reset password.
  • POST /admin-login: Admin login.
  • POST /admin-logout: Admin logout.
  • POST /register: User registration.
  • POST /resend-verification-code: Resend verification code.
  • POST /verify: Verify user.
  • GET /admin/users/:id: Get admin user by ID.
  • DELETE /admin/users/:id: Delete admin user by ID.
  • POST /login: User login.
  • POST /logout: User logout.
  • POST /upload: Upload file.
  • GET /user-documents: Get user documents.
  • GET /documents: Get documents.
  • GET /files/:fileName: Get file by name.
  • GET /download/:fileName: Download file by name.
  • DELETE /documents/:id: Delete document by ID.
  • POST /send-email: Send email.
  • POST /update-profile: Update user profile.
  • POST /admin-profile/:userId: Update admin profile by user ID.
